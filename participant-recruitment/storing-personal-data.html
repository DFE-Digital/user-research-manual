Storing personal data

User researchers store digital files in SharePoint and Google shared drives.

Your team might already have this set up. If not, you’ll need to choose a place to keep participants’ personal data and request access to set it up. You’ll also need to make sure you carefully manage who accesses the files.

Choose a platform to store personal data

As a user researcher at DfE, you’ll use Google Drive and SharePoint to store files depending on how your team is set up. The platform you choose should be decided by your team. They might already be using a certain platform, so check with them before requesting anything new.

The Data Protection Office and security team at DfE recommend using SharePoint to store personal data. You should keep in mind that if you have contractors on your team they may struggle to access SharePoint.

You can choose one platform for files that have personal data in them and another for files without personal data, for example, project files.

SharePoint folder for user researchers

A new process is being developed where you’ll be able to choose to keep participant personal data in the SharePoint workplace for user researchers. With SharePoint, all personal data is kept in one secure place – this mitigates risk by making it easier to manage.

Research Operations have worked closely with the Knowledge and Information Management Team (KIM) to make it easier for you to manage personal data in SharePoint, including moving, setting permissions and deleting.

Using SharePoint helps keeps participant personal data safe because:

  • you can only access this with an invitation link
  • you’ll only be able to see the folders you’ve given access to
  • nobody else will be able to access the data you store in SharePoint without permission
  • you can use retention labels to make sure your data is automatically deleted at the end of a retention period - for example, if you set a retention label of ‘2 years’ the file will be automatically deleted after 2 years

    Read about retention periods for keeping personal data.

    Google shared drives

    You may choose to keep personal data in Google Drive.

    Unlike with Sharepoint, there’s no centrally managed folder (by DfE) for user researchers in Google Drive. You’ll need to create your own folders.

    If you use Google Drive, Research Operations will not be able to see your folders or help you with managing access, so there is an increased risk to managing personal data. Do not add Research Operations colleagues to your folders because the way they are set up on the platform means there are too many for them to monitor.

    The folder that you set up for personal data should be a Google shared drive.

    Request access to shared drives

    Fill out the new requests form or contact the digital tools team for more information. Anyone who needs access to the shared drive will need to have a digital.education.gov.uk account and it may take up to 48 to get access.

    Do not contact individuals from the team directly, always use the #digital-tools-support Slack channel or email digital-tools-support@digital.education.gov.uk. This helps them manage their workload and get to your requests faster.

    Manage who accesses the files

    Only give others access to personal data that has not been anonymised on a ‘need to know’ basis.

    Where possible, user researchers should be the only people accessing personal data. As a user researcher you should limit access to the data to the smallest number of people possible.

    Give access to DfE information asset owners

    DfE needs to keep up to date records of all the data it stores. Places where data is stored are called ‘information assets’. If they contain personal data, some examples of information assets are:

  • individual documents
  • folders containing multiple documents
  • a recruitment panel
  • an online service

    All information assets should be recorded on the DfE information asset register.

    Information asset owners (IAOs) are accountable for the security of data at DfE and they need to have access to folders containing personal data. IAOs are sometimes supported by information asset managers (IAMs). Some teams will have an IAM who is responsible for all of the personal data collected in user research.

    Giving access to other people

    Where it is not manageable for one researcher to conduct research and also manage all of the data compliantly, access can be given to someone who is not a user researcher, an IAO or an IAM, to help with data management. This person must understand UK GDPR and how it relates to good data management.

    Anyone other than your team, an IAO or IAM can only be given access to personal data to help with compliance, never just to have access to research.
